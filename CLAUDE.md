### Thinking Mode vs Writing Mode

This vault is designed for AI-assisted **thinking**, not just AI-assisted **writing**.

**Thinking Mode (Research & Exploration):**
- Claude asks questions to understand your goals
- Searches existing notes for relevant content
- Helps make connections between ideas
- Maintains a log of insights and progress

**Writing Mode (Content Creation):**
- Generates drafts based on your research
- Helps structure and edit content
- Creates final deliverables

**Usage Recommendations:**
1. **Start in Thinking Mode** - Resist the urge to generate content immediately
2. **Explore thoroughly** - Ask questions, search existing notes, build connections
3. **Confirm direction** - Once goals and approach are clear, switch to Writing Mode
4. **Save everything** - Chat logs, fragments, partial thoughts all have value

**Example Prompts:**
```
I'm starting a new project about [topic].
I'm in thinking mode, not writing mode.
Please search my vault for any relevant existing notes,
then help me explore this topic by asking questions.
```

### The PARA Method Explained

**Projects** - Have a deadline and specific outcome
- **Definition:** Time-bound efforts with clear goals
- **Examples:** "Q1 2025 Job Search Strategy", "ESP32 Autodiary Tool", "Novel Chapter 1"
- **Location:** `01_Projects/`
- **Characteristics:** Clear start and end dates, measurable outcomes
- **Lifecycle:** Initiation â†’ Research â†’ Development â†’ Review â†’ Completion â†’ Archive

**Areas** - Ongoing without an end date
- **Definition:** Ongoing responsibilities requiring continuous attention
- **Examples:** "Health", "Finances", "Career Development", "Professional Network"
- **Location:** `02_Areas/`
- **Characteristics:** Define your roles and commitments, need regular maintenance
- **Maintenance:** Weekly scans, monthly reviews, quarterly deep dives

**Resources** - Topics of ongoing interest
- **Definition:** Reference materials and evergreen notes knowledge base
- **Examples:** "AI Product Management", "Hardware Design Patterns", "Writing Techniques"
- **Location:** `03_Resources/`
- **Characteristics:** Organized by topic not source, evergreen not ephemeral
- **Principles:** One concept per note, self-contained understanding, heavily linked

**Archive** - Inactive items
- **Definition:** Completed or inactive projects preserved for future reference
- **Examples:** Completed projects with summaries, abandoned ideas
- **Location:** `04_Archive/`
- **Characteristics:** Preserves institutional memory, past informs future

**Decision Criteria:**
```
Has deadline and specific outcome?  â†’ Project (01_Projects)
Ongoing without end date?           â†’ Area (02_Areas)
Reference material or knowledge?    â†’ Resource (03_Resources)
Completed or inactive?              â†’ Archive (04_Archive)
```


### Gemini Vision (MCP Configured)

Google Gemini Vision is configured via MCP server, enabling Claude Code to directly process images and documents without manual API setup.

**Features:**
- **Direct image analysis:** Claude sees the actual image, not your description
- **PDF text extraction:** Full document text without copy-pasting
- **Bulk processing:** Analyze multiple screenshots or documents at once
- **Smart organization:** Auto-generate filenames based on image content
- **Comparison tasks:** Compare before/after screenshots, designs, etc.

**Why this matters:** Instead of describing "a screenshot showing an error message", Claude Code directly sees and reads the error. Perfect for debugging UI issues, analyzing charts, or processing scanned documents.

**Usage:**
```
Analyze this image: 05_Attachments/circuit-diagram.png
What components are shown?
```

### Obsidian Vault Management (MCP Configured)

Obsidian MCP server is configured, enabling direct vault operations, advanced search, and link management without manual file operations.

**Features:**

**1. Note Operations**
- **Create notes** - Generate new notes with frontmatter and content
- **Read notes** - Access note content and metadata
- **Update notes** - Modify existing notes (replace or append)
- **Delete notes** - Remove notes safely

**2. Search & Query**
- **Content search** - Find notes by text, tags, or properties
- **Date-based search** - Filter by creation or modification date
- **Property search** - Query by frontmatter metadata
- **List all notes** - Get vault inventory

**3. Tag Management**
- **Add tags** - Tag notes for organization
- **Update tags** - Modify tag lists (merge or replace)
- **Remove tags** - Clean up tags
- **List all tags** - See tag taxonomy

**4. Link Management**
- **Backlinks** - Find notes linking to current note
- **Outgoing links** - See what current note references
- **Broken links** - Identify and fix dead links
- **Orphaned notes** - Find notes with no connections

**Why this matters:** Automate vault maintenance, ensure PARA structure compliance, batch process notes, and maintain knowledge graph health - all programmatically through Claude Code.

**Usage Examples:**
```
Search all notes tagged with #job-search and list them by date

Find all notes in 01_Projects that haven't been updated in 30 days
```

## â“ Troubleshooting

### Claude Code Can't Find My Notes

**Symptom:** Claude says it can't find a note or file

**Solutions:**
- Make sure you're running Claude Code from the vault root directory
- Check file permissions: `ls -la [file path]`
- Verify markdown files have `.md` extension
- Check filename spelling and case

### Git Conflicts

**Symptom:** Conflicts when pulling or pushing

**Solutions:**
- Always `git pull` before starting work
- Commit frequently with clear messages
- Use branches for experimental changes
- When conflicts occur:
  ```bash
  git status  # See conflicted files
  # Manually edit conflicted files
  git add .
  git commit -m "Resolve merge conflict"
  ```

### Attachments Management Issues

**Symptom:** Attachment files are messy, can't find files

**Solutions:**
- Run `pnpm attachments:create-organized` to set up folders
- Use `pnpm attachments:orphans` to find orphaned files
- Keep attachments under 10MB for Git
- Use Git LFS for large files or external storage
- Regular cleanup: `pnpm attachments:sizes` to find large files

### Gemini Vision Not Working

**Symptom:** Image analysis commands fail or no response

**Solutions:**
- Check environment variable: `echo $GEMINI_API_KEY`
- Verify API key validity (login to Google AI Studio)
- See `.claude/mcp-servers/README.md` for full configuration
- Restart Claude Code session
- Check API quota: visit Google AI Studio dashboard

### Obsidian MCP Issues

**Symptom:** Can't read/write notes, search fails, or operations timeout

**Solutions:**
- Verify MCP server is running and connected to vault
- Check vault path is correct in MCP configuration
- Ensure note paths use forward slashes (`/`) not backslashes
- For search failures, simplify query - try basic text search first
- If notes aren't found, verify file has `.md` extension
- For permission errors, check file/folder permissions: `ls -la`

**Common Issues:**
- **"Note not found"**: Check exact path and spelling, include `.md`
- **"Invalid path"**: Paths are relative to vault root, no leading `/`
- **Search returns nothing**: Check tag format (`#tag` vs `tag`)
- **Links broken**: Verify wiki-link format `[[note-name]]` not `@note-name`

### Chromium Browser Automation Issues

**Symptom:** Navigation or content extraction fails

**Solutions:**
- Verify MCP server is running: Check Claude Code MCP status
- Some websites may block automation - try different sites to test
- Check if JavaScript is required - Chromium handles this automatically
- For complex sites, break down navigation into smaller steps
- Use screenshots to debug - verify what the browser sees
- Check console errors with network monitoring tools

**Fallback:** If browser automation fails consistently, Firecrawl MCP can be used as a simpler alternative (requires API key configuration)

### Performance Issues

**Symptom:** Vault is slow to respond, search is slow

**Solutions:**
- Check vault size: `pnpm vault:stats`
- Clean up large attachments: `pnpm attachments:sizes`
- Archive old projects to `04_Archive/`
- Delete unnecessary files
- Consider splitting into multiple vaults (work/personal)

### Can't Find a Feature

**Symptom:** Remember there's a command but can't find it

**Solutions:**
- Check this file for complete command list
- Look in `.claude/commands/` directory
- Check `package.json` scripts section
