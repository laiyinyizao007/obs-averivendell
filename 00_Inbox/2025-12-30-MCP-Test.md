## 配置历程

### 方案演进
1. **初始方案**: Docker + HTTP 传输
   - 遇到问题：npm 包名错误、环境变量配置、Linux Docker 网络、HTTP vs HTTPS
2. **最终方案**: stdio 传输（直接进程通信）
   - 优势：配置简单、无 Docker 开销、更稳定可靠

### 关键发现
- Obsidian REST API 使用 HTTPS（端口 27124），不是 HTTP
- obsidian-mcp-server 主要设计用于 stdio 模式
- stdio 模式比 HTTP server 模式更简单可靠
